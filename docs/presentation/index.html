DIVD Event Terminal

type an idea  →  get a receipt

a hacker-terminal kiosk on a Raspberry Pi 5

Welcome. This is the DIVD Event Terminal. In one sentence: you type an idea and it prints you a receipt. It's a booth gadget dressed up as a movie hacker's terminal, all running on a single Raspberry Pi 5. I'll show you what it does, then walk through the hardware.

It prints
your ideas.

Type at the booth. Hit Enter. Walk away with paper.

The whole point in one line: it prints people's ideas. A visitor types something at the booth, hits Enter, and walks away holding a printed receipt with the DIVD logo on it — a little souvenir.

The loop

type→ browser→ Django→ save→ print→ ✂ cut

every 4th receipt → bonus ASCII art or a cat photo

Here's the path every idea takes. You type in the browser; it posts to our Django app on the Pi; we save it to a database so we keep every idea; then we render a receipt and send it to the printer, which prints and auto-cuts. A fun touch: every fourth receipt also spits out bonus ASCII art or a dithered cat photo.

4 parts.

brain · screen · printer · lights

↓ press down to walk the hardware

The hardware is just four parts: the brain, the screen, the printer, and the lights — plus a keyboard. Let's take them one at a time. I'll page downward through each.

1

Raspberry Pi 5

the brain

tiny Linux computer · runs everything · boots straight into the kiosk

no desktop · no login · no mouse

The brain is a Raspberry Pi 5 — a credit-card-sized Linux computer. It runs everything: the web app, the database, the display, the printer, the lights. It boots straight into the kiosk — no desktop, no login screen, no mouse. Power it on and it's ready.

2

Screen

the hacker UI

full-screen Chromium · fake tmux + htop · glitching DIVD logo

attackersdefenders

locked down — Alt-Tab / F11 / escape all blocked

Just a regular HDMI monitor, but locked down. The Pi runs Chromium full-screen so it only ever shows our page: fake tmux panes, a fake htop with red attackers versus green defenders, and a glitching DIVD logo. Escape keys like Alt-Tab and F11 are blocked, so curious hands can't break out at a public booth.

3

Printer

Epson TM-T20III · thermal · USB

no ink · the whole receipt is drawn as an image

DIVD logo · your idea · IDEA No. NNNN

An Epson TM-T20III — the same thermal receipt printer shops use. No ink; it burns the image onto heat-sensitive paper. It connects over USB. We don't send it text — we draw the whole receipt as an image: DIVD logo, the idea, and an "Idea number" counter.

18 styles

/big /tiny /comicsans /qr /barcode

/glitch /wanted /leet /mirror /lacewing

type a slash-command before your idea

There are eighteen print styles hidden behind slash-commands. Type slash-big, slash-tiny, slash-comicsans, slash-qr, slash-glitch and so on before your idea and it changes how it prints. Slash-lacewing prints the DIVD manifesto. It rewards people who poke at it.

4

LED strip

15× WS2812B · Knight-Rider sweep

print → green flash · image → rainbow

volume keys = speed & brightness · never breaks a print

The lights are a strip of fifteen addressable LEDs. At rest they do a Knight-Rider sweep in DIVD yellow. Events change the colour — a print flashes green, an image egg cycles a rainbow. The keyboard's volume keys control speed and brightness. It's deliberately fail-soft: if the lights break, printing still works.

How it all plugs in

PartConnectionOn the Pi
ScreenHDMIHDMI0
KeyboardUSBUSB
PrinterUSB/dev/usb/lp0
LED strip DINSPI0 MOSIpin 19 · GPIO 10
LED power+5V / GNDpin 2 / pin 6

LEDs ride the SPI bus — one wire of data, no extra driver chip

Quick wiring overview. Screen is HDMI. Keyboard and printer are USB — the printer shows up as dev-usb-lp0. The LED strip's data line is on physical pin 19, GPIO 10, which is the SPI bus, with 5 volts and ground from the header. One data wire, no extra driver chip.

Inside the box

Epson printer · Raspberry Pi 5 + cabling · one hinged case

And here's the back, with the lid off — the actual guts. The big black box is the Epson thermal printer. Down in the corner is the Raspberry Pi 5 with its nest of cables, and the whole thing is bolted into a single hinged case.

ToF sensor

VL53L0X time-of-flight · on I²C (SDA pin 3 · SCL pin 5)

the plan: wave your hand to drive the lights

✗ dropped — flaky & fiddly

→ replaced by the keyboard's volume keys

Originally we wanted a hand-wave control: a VL53L0X time-of-flight distance sensor on the I2C pins, so you'd wave your hand to drive the lights. It turned out flaky and fiddly to mount, so we dropped it and used the keyboard's volume keys instead — simpler and rock-solid.

Don't break out.

it's a public booth — people will poke at it

Alt-TabAlt-F4F11 right-clickCtrl-Alt-F‹n›

all blocked → screen flashes red

JS key-guard · cage = no window manager · VT-switch off (srvrkeys:none + NAutoVTs=0) · no cursor · admin is password-gated

Because it's a public booth, people will try to escape the kiosk and get to the desktop — so we defend against it on several layers. A JavaScript guard traps the obvious key combos — Alt-Tab, Alt-F4, F11, right-click — and flashes the screen red instead. Below the browser, there is no desktop to escape to: cage is a bare Wayland kiosk with no window manager. The Ctrl-Alt-F-key virtual-terminal switch is disabled at the keyboard layer with srvrkeys:none and in logind with NAutoVTs zero, so you can't drop to a text console. The cursor is hidden, and the hidden admin functions — shutdown, reboot, wifi — are all password-gated. Defense in depth, which is very on-brand for DIVD.

Under the hood

Django 6 SQLite Pillow cage + Chromium systemd

one Pi · every part its own service

Software-wise: Django 6 for the web app, SQLite for storage, Pillow to draw the receipts, cage plus Chromium for the kiosk display, and systemd to tie it together. Each part is its own service, so any one can restart without taking down the rest.
