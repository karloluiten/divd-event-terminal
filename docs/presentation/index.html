What is this thing?

A booth attraction with a job:

  • A visitor walks up, types an idea, hits Enter.
  • It prints on a real receipt printer — DIVD logo, the idea, an IDEA No. NNNN counter.
  • The screen looks like a movie hacker's terminal: tmux panes, a fake htop, a glitching DIVD logo.
  • Loads of easter eggs reward people who poke at it.

The core loop

Visitor types idea → Browser (kiosk) → Django /submit → Save to database → Render receipt (Pillow) → Thermal printer → paper cut ✂

Every print also fires a tiny message to the LED strip so the lights flash green. Every 4th receipt prints bonus ASCII art or a dithered cat photo.

1 Raspberry Pi 5 — the brain

  • Credit-card-sized Linux computer (Raspberry Pi OS, 64-bit).
  • Runs the whole stack: web server, database, display, printer, LEDs.
  • Boots straight into the kiosk — no desktop, no mouse, no login screen.
  • Everything lives at /opt/divdprint and runs as the unprivileged divd user.

On boot it:

  • starts the Django app (gunicorn on port 8000)
  • launches a full-screen browser pointed at it
  • prints a hello, world! receipt to prove the printer is alive
  • starts the LED scanner

2 The screen — kiosk display

  • A plain HDMI monitor — but locked down so it only ever shows our page.
  • The Pi runs cage (a minimal Wayland kiosk compositor) which launches Chromium full-screen at localhost:8000.
  • No window borders, no address bar, no cursor — it looks like a bespoke appliance, not a web page.
Escape defenses: Alt-Tab, Alt-F4, F11 and Ctrl-Alt-F-keys are all blocked — try to break out and the screen flashes a cheeky red warning. The kiosk is meant to survive curious hands at a public booth.

What's on the screen

A fake "movie hacker" desktop made of four tmux-style panes:

  • Left — terminal: where visitors type their idea, plus a slash-command menu of print styles.
  • Top-right — fake htop: red "attacker" processes vs. green Lacewing "defender" processes, live counters.
  • Bottom-right — glitching DIVD logo with random takeover animations.
  • Status bar: live ideas:N counter + clock.

All decorative — the real work is one text box and an Enter key.

3 The printer — Epson TM-T20III

  • A thermal receipt printer — same kind shops use. No ink; heat darkens special paper.
  • Connects over USB; the Pi sees it as /dev/usb/lp0.
  • We don't send text — we draw the whole receipt as an image with Pillow, then send raw ESC/POS commands to print and auto-cut it.

Every receipt has:

  • a DIVD logo header (drawn pixel-by-pixel)
  • the visitor's idea
  • IDEA No. NNNN counter + divd.nl

No date/time — the booth is offline by design.

18 easter-egg print styles

Type a /slash command before your idea to change how it prints:

  • /comicsans · /big · /tiny
  • /shout · /leet · /mirror
  • /upsidedown · /redacted
  • /wanted — old-west poster
  • /qr · /barcode — scannable
  • /glitch · /spooky · /retro
  • /fancy · /wingdings · /ascii
  • /lacewing — the DIVD manifesto

Whatever the style, the idea is always saved to the database.

4 The LED strip — WS2812B

  • 15 individually-addressable RGB LEDs (~50 cm), wired to the Pi's SPI pin (pin 19).
  • Idles as a Knight-Rider scanner — a DIVD-yellow blob sweeping back and forth.
  • Reacts to events: a print flashes it green, an image-egg cycles a rainbow, the "ransomware" prank goes red, etc.

Physical control = the keyboard's media keys:

  • Vol +/− → scanner faster / slower
  • Shift+Vol → brighter / dimmer
  • Mute → pause

Settings persist across reboots.

The lights are fail-soft — if anything goes wrong with them, printing is never affected.

The LED strip is owned by one dedicated process; the web app just sends it a quick "play this effect" note and moves on — so the lights can never slow down a print.

Under the hood

LayerTech
Web appDjango 6 + gunicorn (Python 3.13)
DatabaseSQLite — every idea, the mode used, errors
Receipt renderingPillow (images) + qrcode / Code128
Displaycage (Wayland) + Chromium kiosk
LEDsAdafruit Blinka + rpi_ws281x
Gluesystemd services, runs on cold boot

Each piece is its own systemd service: web, display, LEDs, volume-keys, boot-receipt — so any one can restart without taking down the rest.
